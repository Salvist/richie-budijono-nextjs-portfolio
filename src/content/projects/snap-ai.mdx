## Context

Snap AI was an independent experiment in making generative image models approachable on mobile and web. Building the interface was only one part of the work: the product also needed authentication, usage limits, storage, asynchronous generation, and content-safety safeguards.

## Product scope

- Generate images from natural-language prompts.
- Transform an existing image using a new prompt.
- Keep generation history tied to authenticated users.
- Coordinate model requests, cloud storage, and usage limits through Firebase.

## The difficult part

After the mobile release, generated content exposed the limits of simple word filtering. I investigated a layered moderation approach using Google Cloud Vision safe-search analysis on generated outputs. The work became a useful lesson in treating AI safety as a system responsibility, not a checkbox on the model request.

## Technical decisions

- **Flutter** for the mobile and web product surfaces.
- **Firebase** for authentication, data, storage, serverless functions, and hosting.
- **Stable Diffusion provider APIs** for image generation.
- **Google Cloud Vision** for output-level safety analysis.
